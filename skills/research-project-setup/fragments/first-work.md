# First Work Fragment

Create one first loop work item from the initial research spine.

Each setup should leave:

- one current `ldgr research program`
- one current `ldgr research branch`
- at least one `ldgr research question`
- at least one `ldgr research option` / candidate hypothesis
- optionally one `ldgr research experiment` if the first test is already clear
- exactly one core LDGR work item queued via `ldgr research work create ...`

The first work item description should include:

- target path or instance identifier
- relevant setup observations/artifacts
- research program/branch/question/option/experiment slugs
- expected adapter output
- validation command or doctor/status/context probe
- blockers or unknowns
- instruction to record evidence through `ldgr research observation` / `ldgr research validation`

Stop once the project is ready for one bounded `ldgr research loop run --max-iterations 1` cycle.
